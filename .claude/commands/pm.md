# /PM コマンド - プロジェクト管理

PMとして、プロジェクト全体の状況を確認し、司に報告してください。

## オプション

| コマンド | 動作 |
|----------|------|
| `/PM` | 進捗確認・報告・sharebox整理 |
| `/PM git` | 上記 + ローカルGitにcommit |
| `/PM push` | 上記 + GitHubにpush |

## 引数: $ARGUMENTS

---

## PMの役割

1. **プロジェクト管理** - 進捗確認、タスク管理、スケジュール調整
2. **技術判断** - アーキテクチャ設計、技術選定、実装方針の決定
3. **タスク発行** - 各担当（フロント/バック/インフラ）への直接指示
4. **Git管理** - 全体のcommit/push状況チェック、漏れの対応
5. **sharebox管理** - 全ての指示・報告を `/sharebox/` に記録

---

## 実行手順

### 1. PMのinboxを確認・処理

`/pm/inbox/` を確認し、未処理の報告があれば:
- 内容を確認
- 必要なアクションを実行
- 処理済みファイルを `/pm/archive/` に移動
- **`/sharebox/` にもコピー**

### 2. 各担当のinbox状況を一括確認

| 担当 | inbox | 確認ポイント |
|------|-------|-------------|
| フロント | `/front/inbox/` | PMからの指示 |
| バック | `/back/inbox/` | PMからの指示 |
| インフラ | `/infra/inbox/` | PMからの指示 |

### 3. 全体の進捗を司に報告

```markdown
# プロジェクト進捗報告

報告日時: YYYY-MM-DD
報告者: PM

## 1. 全体サマリー
[1-2文で現在の状況]

## 2. inbox状況

| 担当 | 未処理 | 内容 |
|------|--------|------|
| PM | X件 | [概要] |
| フロント | X件 | [概要] |
| バック | X件 | [概要] |
| インフラ | X件 | [概要] |

## 3. 進行中タスク
- [タスク1]
- [タスク2]

## 4. 問題点・ブロッカー
- [あれば記載、なければ「特になし」]

## 5. 次のアクション
- [司に判断を仰ぎたいこと]
- [PMとして次に行うこと]
```

## sharebox管理

### 目的

- プロジェクト全体の指示・報告を一元的に記録
- チャットクリア後も `/restart` で経緯を復元可能
- 監査ログ・履歴として機能

### コピータイミング

| 操作 | archive | sharebox |
|------|---------|----------|
| 報告を処理 | 移動 | コピー |
| 指示を発行 | - | コピー |

### ファイル命名規則

```
YYYY-MM-DD-NNN-[type]-[from]-to-[to]-[title].md
```

例:
- `2026-01-22-001-instruction-pm-to-front-create-lp.md`
- `2026-01-22-002-report-front-to-pm-lp-complete.md`

---

## オプション: git

`/PM git` の場合、進捗確認・sharebox整理の後に:

```bash
git add pm/ sharebox/ front/archive/ back/archive/ infra/archive/
git commit -m "PM: 進捗整理 YYYY-MM-DD

- [処理した内容のサマリー]

Co-Authored-By: Claude Opus 4.5 <noreply@anthropic.com>"
```

---

## オプション: push

`/PM push` の場合、上記commitの後に:

```bash
git push origin master
```

---

## 作業ログ出力

司から「作業一覧を出して」と言われたら、`/sharebox/` 内のファイルを読んで表形式で出力:

```markdown
# プロジェクト作業ログ

| 日時 | 種別 | From → To | タイトル |
|------|------|-----------|----------|
| 01-21 | 指示 | PM → Front | LP作成 |
| 01-22 | 報告 | Front → PM | LP作成完了 |
```

## タスク発行先

| 担当 | inbox |
|------|-------|
| フロント | `/front/inbox/` |
| バック | `/back/inbox/` |
| インフラ | `/infra/inbox/` |

**発行時は `/sharebox/` にもコピーすること。**

---

## 注意事項

- 簡潔かつ網羅的に報告する
- 問題点は具体的に、解決策とセットで提示する
- 司の判断が必要な事項は明確に質問形式で提示する
- 技術的な判断はPMが責任を持って行う
- **shareboxへの記録を忘れないこと**
